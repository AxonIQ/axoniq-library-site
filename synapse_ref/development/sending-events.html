Sending Events

Synapse provides three ways to send events to Axon Server:

  • an endpoint /v1/context/[contextName]/events/[eventName] (raw endpoint), where the content of the HTTP request is the payload of the event. All the other optional properties can be provided through HTTP headers.

  • an endpoint /v1/context/[contextName]/events (message endpoint), where the content of the HTTP request is a JSON message containing all the information for one event.

  • an endpoint /v1/context/[contextName]/events (list-of-messages endpoint), where the content of the HTTP request is a list of JSON messages each element containing all the information for one event.

The difference between the second and third form is in the content type provided. For the message endpoint this is ` application/json, for the list of messages it is application/vnd.axoniq.event.list+json.

Raw endpoint

The raw endpoint has the following required headers:

  • Content-Type: the serialization type for the context

It has the following optional headers:

  • AxonIQ-AggregateId:

  • AxonIQ-AggregateType:

  • AxonIQ-SequenceNumber:

  • AxonIQ-DateTime:

A sample event:

POST http://synapse:8080/v1/contexts/default/queries/local.application.client.Event
Content-Type: application/json
AxonIQ-AggregateId: 901aa5ce-b281-4788-97f2-2be7f236dde6
AxonIQ-AggregateType: Aggregate
AxonIQ-SequenceNumber: 0
AxonIQ-DateTime: 2022-09-22T21:37:00.000+00:00

{
   "id": "901aa5ce-b281-4788-97f2-2be7f236dde6",
   "text": "New customer created"
}

This sends the JSON body as an event to Axon Server. It will be stored in the event store in the default context as this is specified in the URL of the request.

Message endpoint

The message endpoint expects a JSON message with all the information for the event. The same request as above now in message form:

POST http://synapse:8080/v1/contexts/default/events
Content-Type: application/json

{
  "payload": {
    "id": "901aa5ce-b281-4788-97f2-2be7f236dde6",
    "text": "New customer created"
  },
  "name": "local.application.client.Event",
  "aggregateId": "901aa5ce-b281-4788-97f2-2be7f236dde6",
  "aggregateType": "Aggregate",
  "sequenceNumber": 0,
  "dateTime": "2022-09-22T21:37:00.000+00:00"
}

The message contains more optional fields:

  • aggregateId

  • aggregateType

  • sequenceNumber

  • dateTime

  • metaData

list of messages endpoint

This is similar to the normal messages endpoint, except that it expects a slightly different input message. The input message contains an items element which is a list of events.

A same request in list of messages form:

POST http://synapse:8080/v1/contexts/default/events
Content-Type: application/vnd.axoniq.event.list+json

{
"items": [
  {
    "payload": {
      "id": "901aa5ce-b281-4788-97f2-2be7f236dde6",
      "text": "New customer created"
    },
    "name": "local.application.client.Event",
    "aggregateId": "901aa5ce-b281-4788-97f2-2be7f236dde6",
    "aggregateType": "Aggregate",
    "sequenceNumber": 0,
    "dateTime": "2022-09-22T21:37:00.000+00:00"
  },
  {
    "payload": {
      "id": "901aa5ce-b281-4788-97f2-2be7f236dde6",
      "text": "Customer moved"
    },
    "name": "local.application.client.Event",
    "aggregateId": "901aa5ce-b281-4788-97f2-2be7f236dde6",
    "aggregateType": "Aggregate",
    "sequenceNumber": 1,
    "dateTime": "2022-09-22T21:37:01.000+00:00"
  }
  ]
}

You can only pass metaData using the message or list of messages endpoint, it is not supported for raw endpoints.
