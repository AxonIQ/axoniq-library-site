Receiving Events

An event handler can be any service that can be invoked using a URL. Synapse can send queries in three ways:

  • event payload as HTTP body (raw), other information may be passed in headers

  • event as a JSON message (message), the JSON message includes additional information which would be passed as headers in the raw form.

  • list of events as a JSON message (list-of-messages), the JSON message contains a list of events, each event contains the payload and additional information which would be passed as headers in the raw form.

Raw endpoint

This type of event handler endpoint gets the event payload as the body of the message.

It gets the following HTTP headers:

  • Content-Type

  • AxonIQ-MessageId

  • AxonIQ-EventName

  • AxonIQ-DateTime

  • AxonIQ-Index

  • AxonIQ-AggregateId

  • AxonIQ-AggregateType

  • AxonIQ-SequenceNumber

Method endpoint

The message endpoint gets the event payload wrapped in a JSON object that also contains the fields that are sent as headers in the raw endpoint. Additionally, the JSON may contain metadata passed with the query.

Handler definition

To make an event handler read events from Axon Server it needs to be registered in Synapse. This can be done through the Synapse user interface, or through the REST interface.

The registration contains the following fields:

  • Event names, a list of names for the event types that this handler can handle, may be empty to receive all events

  • Client ID, unique id for the handler application instance

  • Component name, logical name for the handler application

  • HTTP Endpoint, the URL of the handler

  • Endpoint Type, specifies whether the handler expects a raw or message request type

  • Endpoint Options, any key/value pairs that will be included in requests to the handler as HTTP headers

  • Batch size, maximum number of events that Synapse sends in one request to the handler, if the endpoint type is list of messages.

  • Start, the global index of the first event to read from the event store, use 0 to start from the first event, use -1 to start from the last event.

A sample JSON request to register a handler:

POST http://synapse:8080/v1/contexts/default/handlers/events
Content-Type: application/json

{
  "names": [
    "local.application.client.Event"
  ],
  "endpoint": "https://client.application.local/v1/message",
  "endpointType": "http-raw",
  "endpointOptions": [
    {
      "key": "string",
      "value": "string"
    }
  ],
  "clientId": "application-name-7c78946494-p86ts",
  "componentName": "application-name",
  "batchSize": 0,
  "start": 0,
  "enabled": true
}

For the operations to update, delete and retrieve handlers, check the http://synapse:8080/swagger-ui.html endpoint.
